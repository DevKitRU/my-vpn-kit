# CONTEXT_HYGIENE

Правила, чтобы `docs/ai_context` не превратился в свалку.

## Главное правило

```text
Контекст должен быть роутером, а не архивом.
```

Карта должна говорить, куда смотреть.

Она не должна хранить весь проект, старые чаты, логи, базы или секреты.

## Лимиты-предупреждения

| Файл | Когда пора сокращать |
| --- | --- |
| `PROJECT_MAP.md` | больше 120 строк |
| `DANGER_ZONES.md` | больше 80 строк |
| `VERIFICATION.md` | больше 120 строк |
| `CURRENT_GOAL.md` | больше 80 строк |
| `SESSION_SUMMARY.md` | больше 150 строк |

Если файл растет, оставь в нем короткий маршрут и вынеси детали в отдельный файл.

## Что хранить в markdown

- карту проекта;
- опасные зоны;
- команды проверки;
- краткий итог сессии;
- ссылки на детали.

## Что хранить в JSONL

- решения;
- точные даты;
- статусы;
- повторяющиеся ошибки;
- findings с evidence-ссылками.

## Что не хранить

- значения токенов;
- `.env`;
- приватные ключи;
- базы данных;
- логи с пользователями;
- полные чаты;
- дампы терминала.

## Ритуал в конце сессии

1. Обновить `SESSION_SUMMARY.md`.
2. Добавить durable decision в `DECISIONS.jsonl`, если решение правда важное.
3. Добавить finding в `optional/FINDINGS.jsonl`, если включен Level 3.
4. Запустить `scripts/check-ai-context.sh`.
5. Убедиться, что секреты и runtime data не попали в diff.
